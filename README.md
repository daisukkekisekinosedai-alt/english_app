# English Pocket Local Plus

Flask + SQLite の英語学習アプリです。ローカルで遊ぶ用に機能追加した版です。

## 追加済み機能

- 10問連続クイズ
- 1問クイズ
- 4択クイズ
- 入力式クイズ
- 発音ボタン
- 単語検索
- カテゴリ管理
- レベル管理
- お気に入り
- 苦手単語
- 学習履歴
- CSVインポート
- CSVエクスポート
- スマホ向け画面

## 起動方法

```powershell
cd C:\Users\81806\python\english_iphone_app_local_plus
pip install -r requirements.txt
python app.py
```

ブラウザで開きます。

```text
http://127.0.0.1:5000
```

## 音声機能

この版ではブラウザの音声読み上げ機能を使って、英単語と例文を聞けます。

追加された音声機能：

- 単語一覧の発音ボタン
- クイズ画面の発音ボタン
- 結果画面の発音ボタン
- ゆっくり再生ボタン
- 例文読み上げ
- 音声を聞いて日本語を答えるリスニング問題
- 10問連続リスニング

リスニング画面：

```text
http://127.0.0.1:5000/listen
```

10問連続リスニング：

```text
http://127.0.0.1:5000/session/start?mode=listen&scope=all&count=10
```

※ 音声はブラウザ側の機能を使うため、端末やブラウザによって声質が変わります。


## 発音改善版について

ブラウザ標準の音声読み上げは、`a` を「エー」と読むなど、英語学習には不自然な読み方になることがあります。

この版では以下を追加しています。

- 読み上げ用テキスト欄
- `a` を文脈に応じて弱形っぽく読む補正
- 英語音声の優先選択
- CSVの `audio_text` / `音声用テキスト` 取り込み

例：

```text
表示する英語: This is a pen.
読み上げ用: This is uh pen.
```

画面には `This is a pen.` と表示し、音声だけ `This is uh pen.` に近い読み方で再生できます。

※ 完璧な発音にするには、本格的なTTS APIや録音音声が必要です。このアプリではローカルで手軽に使える範囲で改善しています。


## 無料TTS音声ファイル版

この版では、無料で音声ファイルを作成できます。

方式は2つあります。

### 1. gTTSでmp3生成

おすすめです。

- 無料
- APIキー不要
- mp3で保存
- ブラウザ読み上げより自然になりやすい
- ネット接続が必要

### 2. Windows音声でwav生成

完全ローカルです。

- 無料
- ネット不要
- wavで保存
- 声質はPCに入っている英語音声に依存

## 使い方

起動後、以下を開きます。

```text
http://127.0.0.1:5000/tts
```

「gTTSで全件生成」を押すと、単語と例文の音声ファイルを作成します。

作成した音声は以下に保存されます。

```text
static/audio/
```

## 発音を直したい場合

単語編集画面で「読み上げ用テキスト」を入れてから、音声ファイルを再生成してください。

例：

```text
表示する英語: This is a pen.
読み上げ用: This is uh pen.
```

画面表示は `This is a pen.` のまま、音声だけ自然寄りにできます。


## 自然な無料TTS版

この版では、無料・APIキーなしで使える `edge-tts` を最優先にして音声ファイルを生成します。

生成順：

1. Edge TTS: 自然寄りのmp3。ネット接続が必要。
2. gTTS: Edgeが失敗した場合のmp3フォールバック。ネット接続が必要。
3. Windows音声: 完全ローカルのwavフォールバック。

## 使い方

```powershell
pip install -r requirements.txt
python app.py
```

音声管理ページ：

```text
http://127.0.0.1:5000/tts
```

おすすめは「Edge自然音声で全件生成」です。

## 発音をさらに直したい場合

単語編集画面の「読み上げ用テキスト」を使います。

例：

```text
表示する英語: This is a pen.
読み上げ用: This is a pen.
```

不自然に読む場合だけ、音声用テキストを少し調整してください。

## 単語データとユーザーの扱い

この版では、単語データは全ユーザー共通です。

- Aさんが追加した単語はBさんも使えます。
- クイズ結果、回答履歴、苦手単語、ランキング集計はユーザーごとに分かれます。
- 同じ英単語を追加しようとした場合は、新規追加せず既存単語の編集画面に移動します。
- 同じ英単語に別の意味を追加したい場合は、日本語欄にカンマ区切りで追記してください。
  例: `請求する, 充電する, 担当させる`

## 本版のユーザー設計

この版では、ユーザー概念を画面上でも明確にしています。

- `/register` でユーザー登録
- `/login` でログイン
- ログインしていない状態では、ホーム・クイズ・単語一覧などの主要画面に入れません
- 画面上部にログイン中ユーザーの名前とアイコンを表示
- `/users` でユーザー一覧を表示
- `/profile` で自分のプロフィールとアイコンを編集
- `/dashboard` はログイン中ユーザーの成績だけを表示
- `/rankings` はユーザー別ランキングを表示
- `/share/today` はログイン中ユーザーの今日の結果を共有

## 単語とユーザーの関係

- 単語は全ユーザー共通です
- Aさんが追加した単語は、Bさんもクイズで使えます
- 単語には「追加者」が表示されます
- お気に入りはユーザー別です
- 苦手単語はユーザー別です
- 回答履歴・テスト結果・共有結果はユーザー別です
- 別ユーザーが追加した単語は削除できません


## 修正メモ

`sqlite3.OperationalError: unknown column "created_by_user_id" in foreign key definition` を修正しました。

原因:
`words` テーブルに追加者IDを持たせる修正の際、外部キー定義だけが先に入り、列定義との整合が崩れていました。

修正:
- `words.created_by_user_id` 列を正しく追加
- 既存DBには起動時に `ALTER TABLE` で自動追加
- `words` テーブル側の壊れやすい外部キー定義は削除


## Game Plus版の追加機能

デイリーミッション以外のゲーム化機能を追加しました。

追加内容:

- 連続学習日数（ストリーク）

- `english.db` はそのまま使えます。
- 経験値・レベル・バッジ・ストリークは既存の回答履歴から自動計算します。
- 新しいテーブルは追加していません。
